This module will have a computer-like background. There are four buttons labeled song 1-4, four more orthogonal arrow buttons, and two buttons labeled "flip" and "submit". Take the number associated with each sbemail and convert it to binary, with zeros added to the left to make it 8 bits long if needed. Concatenate these to get a 32-bit string, and perform the operation associated with the background on each pair of bits (i.e. the 1st and 2nd bit, the 3rd and 4th bit, etc.). Arrange the resulting 16 bits into a 4x4 grid in reading order to get the initial grid.

Appendix M: Additional Notes

Here other things to keep in mind when making a manual page:

  • There are comments /* ... */ in the CSS code in the <style> blocks containing instructions. Follow those instructions before finishing your manual.
  • Normal and Needy module manual pages are formatted similarly. Widget manual pages are appendices and are formatted as such.
  • Each manual or appendix page should have a <div class="page-footer... at the bottom of the page, updated accordingly.
  • Don't be afraid to ask for help if needed!
  • A few things to reiterate:
    • Make sure to never forget closing tags (the ones with a / in them).
    • For the most part, HTML versions of bolding and underlining is recommended.
    • Italics are not used in vanilla manuals besides flavor text and references to appendices, and therefore is not included above.
